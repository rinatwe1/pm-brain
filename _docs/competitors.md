# PM Brain — Competitive Landscape

## Direct / Closest

### AI-SHIPR (verve-pm) — github.com/Yaniv-Git/AI-SHIPR
- 20+ PM skills for Claude Code
- Free, open source
- **Memory: zero** — each session is stateless
- **Our relationship:** complementary, not competing. PM Brain is the memory layer that makes AI-SHIPR skills work better over time. A PM could use both.
- **Our advantage:** AI-SHIPR gives you the verbs (discover, create, verify). PM Brain gives you the memory. Two different jobs.

### Squad AI — squadai.io
- Web-based PM platform with "foresight engine"
- AI-powered, not Claude-native
- **Memory: no** — not longitudinal
- **Our advantage:** PM Brain lives in the PM's IDE. Zero context-switching. Claude-native.

### Pawel Huryn's CLAUDE.md approach (Product Compass newsletter)
- 3 CLAUDE.md blocks: Knowledge Architecture + Decision Journal + Quality Gate
- Manual — the PM copy-pastes snippets into CLAUDE.md
- No install, no structure, no skills
- **Our relationship:** PM Brain IS this approach, productized. We give it structure, skills, agents, and a knowledge base. Pawel validated the concept — we built the tool.

## Key Insight

No tool in the market gives Claude **longitudinal, product-specific memory** that:
1. Accumulates automatically over time
2. Lives in the PM's IDE (not a separate tool)
3. Is structured (not just a chat history)
4. Is versioned and shareable via git
5. Evolves — hypotheses become rules, decisions get superseded

The gap is real. The closest anyone came was Pawel Huryn's manual CLAUDE.md approach — which validated the concept but didn't productize it.
